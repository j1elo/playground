Envia una serie de comandos a un servidor del cluster.

La idea de esta funcion es ser ejecutada en paralelo usando varios hilos de ejecucion, y poder mandar asi un conjunto de comandos a todos los servidores del cluster simultaneamente.

int petic_dregister ( int  sockCliente,
char *  serv 
)

Notifica a todas las maquinas del cluster de que hay un nuevo miembro.

El nuevo dserver es el que tiene la IP y puerto suministrados como argumento. Les envia a todos los servidores, en paralelo (usando un hilo de ejecucion para cada uno), una nueva lista de servidores registrados en el cluster. Para ello emplea la funcion fComCluster() y el comando "nueva_lista".

Parámetros:
sockClientesocket de comunicacion con el cliente.
servservidor nuevo en el cluster.
Devuelve:
0 = OK, -1 = Error.
Respuesta al cliente: servidores registrados (sin incluir al nuevo) y sus IPs y puertos: "<errno> <N> <ip1:puerto1> ... <ipN:puertoN>\n".

Elimina un servidor del cluster.

Elimina del cluster el servidor con la IP y puerto especificados. Lo contrario a petic_dregister(). Les envia a todos los servidores, en paralelo (usando un hilo de ejecucion para cada uno), una nueva lista de servidores registrados en el cluster. Para ello emplea la funcion fComCluster() y el comando "nueva_lista".

Devuelve:
0 = OK, -1 = Error.
Respuesta al cliente: "<errno>\n".

Abre el fichero solicitado.

Crea una copia del fichero y abre la copia; de este modo se logra mantener la semantica de sesion, por la que cada hilo tiene acceso a sus propios cambios del fichero, pero no a los de los demas hilos.

Parámetros:
sockClientesocket de comunicacion con el cliente.
pathruta al fichero que se debe abrir.
modemodo de apertura del fichero.
numCopianumero de copia que le corresponde al fichero.
[out]modoAccesosale el modo de acceso del fichero.
Devuelve:
el puntero FILE* al fichero abierto. NULL si error.
Respuesta al cliente: "<errno>\n".

Modifica el indicador de posicion de fichero.

Parámetros:
sockClientesocket de comunicacion con el cliente.
pFichpuntero al fichero.
argOffsetargumento indicando el offset a desplazar a partir del origen.
argOrigenargumento indicando la posicion de origen del desplazamiento. Debe ser uno de los tres valores definidos en stdio.h:
  • SEEK_SET para el inicio del fichero.
  • SEEK_CUR para la posicion actual del fichero.
  • SEEK_END para el final del fichero.
Devuelve:
0 = OK, -1 = Error.
Respuesta al cliente: "<errno> <posicion actual>\n".
